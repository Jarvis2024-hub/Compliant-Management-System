## Backend Requirements

### Core Software

**PHP 8.x or higher**
- Required for running the backend API
- Must have CLI access (command-line interface)
- XAMPP's PHP installation works fine: `C:\xampp\php\php.exe`
- Verify installation: `php --version`

**MySQL 8.x or higher**
- Database management system
- Can be installed via XAMPP, WAMP, or standalone
- XAMPP is recommended for ease of use

**XAMPP (Recommended)**
- Provides both PHP and MySQL
- Includes phpMyAdmin for database management
- Download: https://www.apachefriends.org/
- Note: We only use XAMPP for MySQL and PHP, not Apache

### System Requirements

**Port Availability**
- Port 8000 must be available for the PHP built-in server
- Port 3306 for MySQL (default)
- Check if port is in use: `netstat -ano | findstr :8000`

**Operating System**
- Windows 10/11 (current setup)
- Linux and macOS also supported with minor path adjustments

### PHP Extensions (Usually included with XAMPP)

The following PHP extensions must be enabled:
- `pdo_mysql` - Database connectivity
- `openssl` - For JWT and encryption
- `json` - JSON encoding/decoding
- `mbstring` - String handling

You can verify extensions in `php.ini` or by running: `php -m`

## Network Requirements

**For Android Emulator Testing**
- Use `http://10.0.2.2:8000` as API base URL
- This is the emulator's alias for host machine's localhost

**For Physical Device Testing**
- Backend server must use `0.0.0.0:8000` (not `localhost:8000`)
- Find your computer's local IP: `ipconfig` (Windows) or `ifconfig` (Linux/Mac)
- Use `http://192.168.x.x:8000` as API base URL in Flutter app
- Ensure device and computer are on the same network

**Firewall Considerations**
- If testing on physical device, Windows Firewall may block port 8000
- Add exception for PHP or temporarily disable firewall for testing

## Troubleshooting Common Issues

**"PHP is not recognized as an internal or external command"**
- Add PHP to system PATH: `C:\xampp\php`
- Restart command prompt after adding to PATH

**"Port 8000 is already in use"**
- Find process using port: `netstat -ano | findstr :8000`
- Kill process or use different port

**"Flutter doctor shows errors"**
- Run `flutter doctor -v` for detailed information
- Most common: Android licenses not accepted
- Fix: `flutter doctor --android-licenses`

**"Cannot connect to backend from Flutter app"**
- Verify backend is running: open `http://localhost:8000` in browser
- Check API base URL in `lib/config/api_config.dart`
- For emulator, must use `10.0.2.2` not `localhost`
